Question 21

Explain about classification problem with an example.

1. Definition

A classification problem is a type of supervised machine learning task in which a model is trained to assign each input data point to one of a set of predefined categories (classes or labels). The model learns the mapping from input features to output labels from labelled training data and then applies this mapping to classify unseen instances.

2. Key Characteristics

  • The output is always categorical (discrete values) — e.g., Yes/No, Spam/Not Spam, Dog/Cat/Bird.
  • It uses a labelled dataset where the correct output is already known for each training example.
  • It is a part of supervised learning — the algorithm learns from ground-truth labels.
  • The model learns decision boundaries from training data and applies them to new, unseen data.

3. Types of Classification

TypeDescriptionExample
Binary ClassificationOnly two possible output classes.Spam / Not Spam; Pass / Fail
Multi-class ClassificationMore than two mutually exclusive classes.Classifying animals → Dog, Cat, Cow
Multi-label ClassificationOne input can belong to multiple classes simultaneously.A movie tagged as Action + Comedy + Drama

4. Working Process of Classification

  1. Data Collection — Gather labelled data (training dataset).
  2. Data Preprocessing — Clean and prepare data: remove noise, handle missing values.
  3. Model Training — Use a classification algorithm to learn patterns from the labelled data.
  4. Model Testing — Evaluate accuracy using a held-out test dataset.
  5. Prediction — Classify new, unseen data into the appropriate category.

5. Example — Email Spam Detection

Email spam detection is a classic binary classification problem. The input features include email content, subject line, sender details, and keyword frequency. The model is trained on labelled emails (Spam / Not Spam) and learns to distinguish between them based on these features.

  • Input: Email content (text, subject, sender details).
  • Process: The model checks for keywords like "free", "offer", "win money" and other patterns.
  • Output: Spam → routed to Spam folder; Not Spam → delivered to Inbox.

6. Common Classification Algorithms

  • Naïve Bayes
  • Logistic Regression
  • Decision Trees and Random Forests
  • Support Vector Machines (SVM)
  • k-Nearest Neighbours (k-NN)
  • Neural Networks

7. Conclusion

A classification problem involves learning a function that maps input features to discrete output labels. It is one of the most fundamental tasks in supervised machine learning, with applications ranging from email filtering and medical diagnosis to image recognition and sentiment analysis. The choice of algorithm depends on the nature of the data, the number of classes, and the performance requirements of the application.

Question 22

Elucidate Logistic Regression and mention the importance of the logit function.

1. Logistic Regression

Logistic Regression is a supervised machine learning algorithm widely used for binary classification problems where the dependent variable can take only two possible outcomes (e.g., 0 and 1, Pass/Fail, Spam/Not Spam). Unlike linear regression, which predicts a continuous unbounded value, logistic regression predicts the probability that a given input belongs to a particular class, keeping the output within [0, 1].

  • It does not directly predict class labels; instead it predicts probabilities, which are then converted into class labels using a threshold value (typically 0.5).
  • It uses the sigmoid function to map any real-valued input into the range (0, 1).
  • It establishes a relationship between the independent variables and the log-odds of the dependent variable.

2. Working Principle

The model computes a linear combination of input features:

z = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ

This value z is then passed through the sigmoid function to produce a probability:

P(Y = 1) = σ(z) = 1 / (1 + e−z)

If P(Y=1) ≥ 0.5, the instance is classified as class 1; otherwise class 0.

3. The Logit Function

The logit function is defined as the natural logarithm of the odds ratio (log-odds):

logit(p) = ln(p / (1 − p))

where p is the probability of the positive outcome. The logit is the inverse of the sigmoid function — it maps a probability in (0, 1) back to the real number line (−∞, +∞).

4. Importance of the Logit Function

PropertyExplanation
Linearises the relationshipThe log-odds are a linear function of the input features: logit(p) = β₀ + β₁x₁ + … + βₙxₙ. This is what makes logistic regression a linear model.
InterpretabilityEach coefficient βᵢ represents the change in log-odds for a one-unit increase in xᵢ, making the model interpretable.
Converts probabilities to unbounded scaleProbabilities are bounded in [0,1]. The logit maps them to (−∞, +∞), allowing standard linear modelling techniques to be applied.
Foundation of log-likelihoodThe logit is used in the derivation of the log-likelihood function, which is maximised during training via gradient ascent.
Prevents trivial outputsWithout the logit transformation, a linear model could predict probabilities outside [0,1], which is mathematically invalid.

5. Advantages and Limitations

AdvantagesLimitations
Simple and computationally efficientCannot handle complex non-linear relationships
Outputs calibrated probability estimatesSensitive to outliers
Interpretable coefficients (log-odds)Assumes a linear relationship between features and log-odds
Works well for linearly separable dataRequires features to be independent (multicollinearity is problematic)

6. Applications

  • Email spam detection
  • Medical diagnosis (disease/no disease)
  • Credit risk assessment (default/no default)
  • Customer churn prediction
  • Sentiment analysis (positive/negative)

7. Conclusion

Logistic Regression is a powerful and widely used classification algorithm that models the probability of a binary outcome using the sigmoid function. The logit function is central to this method — it transforms probabilities into log-odds, enabling a linear relationship between features and the output. This transformation allows logistic regression to be both mathematically valid and interpretable, forming the backbone of many real-world predictive systems.

Question 23

Elaborate in detail about performance metrics in classification problems.

Performance metrics in classification problems are used to evaluate how effectively a machine learning model predicts class labels by comparing predicted values with actual values. These metrics are primarily derived from the confusion matrix.

1. Confusion Matrix

The confusion matrix is a table that summarises the prediction results of a classification model. It records four values:

TermFull FormMeaning
TPTrue PositiveCorrectly predicted positive cases.
TNTrue NegativeCorrectly predicted negative cases.
FPFalse PositiveIncorrectly predicted as positive (Type I Error).
FNFalse NegativeIncorrectly predicted as negative (Type II Error).

2. Accuracy

Accuracy measures the overall correctness of the model — the proportion of all predictions that were correct.

Accuracy = (TP + TN) / (TP + TN + FP + FN)

It may give misleading results when the dataset is imbalanced (e.g., 95% negative class).

3. Precision

Precision measures how many of the predicted positives are actually correct. It answers: Of all instances the model labelled positive, how many truly were?

Precision = TP / (TP + FP)

High precision → few false positives. Critical when false alarms are costly (e.g., spam detection).

4. Recall (Sensitivity)

Recall measures how many of the actual positives were correctly identified. It answers: Of all instances that were truly positive, how many did the model catch?

Recall = TP / (TP + FN)

High recall → few missed positives. Critical when missing a positive is dangerous (e.g., disease diagnosis).

5. F1 Score

The F1 Score is the harmonic mean of Precision and Recall. It provides a single balanced metric, especially useful when class distribution is unequal.

F1 = 2 × (Precision × Recall) / (Precision + Recall)

6. Specificity

Specificity measures how well the model identifies actual negatives.

Specificity = TN / (TN + FP)

7. ROC Curve (Receiver Operating Characteristic)

The ROC curve is a graphical representation of the trade-off between the True Positive Rate (Recall) and the False Positive Rate across different classification thresholds.

  • X-axis: False Positive Rate = FP / (FP + TN)
  • Y-axis: True Positive Rate (Recall) = TP / (TP + FN)
  • A curve closer to the top-left corner indicates better performance.
  • Helps in selecting the best threshold for classification.

8. AUC (Area Under the Curve)

AUC represents the area under the ROC curve. It measures the overall ability of the model to distinguish between classes across all thresholds.

  • AUC = 1.0 → Perfect model
  • AUC = 0.5 → Random guessing (no discriminative ability)
  • Higher AUC → better model performance

9. Summary Table

MetricFormulaFocusBest Value
Accuracy(TP+TN) / TotalOverall correctness1.0
PrecisionTP / (TP+FP)Quality of positive predictions1.0
RecallTP / (TP+FN)Coverage of actual positives1.0
F1 Score2×P×R / (P+R)Balance of precision & recall1.0

10. Conclusion

Performance metrics allow practitioners to evaluate classification models beyond simple accuracy. Precision, Recall, F1, and AUC each capture different aspects of model quality. The choice of metric depends on the application: recall is prioritised when missing positives is costly, precision when false alarms are costly, and F1 when balance is required. The ROC curve and AUC provide a threshold-independent view of overall model discrimination ability.

Question 25

Explain about the gradient descent algorithm and its variants.

1. Definition

Gradient Descent is an optimisation algorithm used in machine learning to minimise a loss (cost) function by iteratively updating model parameters in the direction of the negative gradient, thereby reducing the error step by step until convergence.

  • The algorithm begins by initialising parameters randomly, then computes the gradient of the loss function with respect to those parameters.
  • Parameters are updated using a learning rate α, which controls the step size of each update.
  • This process repeats until convergence — i.e., until the cost function reaches (or approaches) its minimum.

2. Basic Idea

Imagine a hill-shaped curve representing the cost function. The goal is to reach the lowest point (minimum). Gradient Descent takes steps downhill by calculating the slope (gradient) at the current position and moving in the opposite direction.

3. Mathematical Formula

The parameter update rule is:

θ = θ − α · (dJ(θ) / dθ)
SymbolMeaning
θModel parameters (weights)
αLearning rate — controls step size
J(θ)Cost function to be minimised
dJ(θ)/dθGradient (slope) of the cost function with respect to θ

4. Steps of Gradient Descent

  1. Initialise parameters (weights) randomly.
  2. Compute the cost function J(θ).
  3. Calculate the gradient (partial derivatives) of J with respect to each parameter.
  4. Update all parameters using the update rule: θ = θ − α · ∇J(θ).
  5. Repeat steps 2–4 until convergence (minimum reached or change is negligibly small).

5. Variants of Gradient Descent

(a) Batch Gradient Descent

Uses the entire training dataset to compute the gradient at each step. Produces stable, accurate updates but is computationally expensive for large datasets.

(b) Stochastic Gradient Descent (SGD)

Updates parameters using only a single randomly selected data point at each step. Much faster but noisy — the cost function fluctuates considerably during training.

(c) Mini-Batch Gradient Descent

Uses a small random subset (mini-batch) of the data at each step. Balances the stability of Batch GD and the speed of SGD. This is the most widely used variant in practice (used in deep learning frameworks by default).

(d) Advanced Variants

  • Momentum — Accumulates a velocity vector in the direction of persistent gradients, helping to accelerate convergence and reduce oscillations.
  • RMSProp — Adapts the learning rate for each parameter by dividing by a running average of recent gradient magnitudes, stabilising updates.
  • Adam (Adaptive Moment Estimation) — Combines Momentum and RMSProp. Maintains both a running mean and a running variance of gradients. Widely used as the default optimiser in deep learning.

6. Comparison of Variants

VariantData Used Per StepSpeedStabilityBest For
Batch GDFull datasetSlowHighSmall datasets
SGD1 sampleFastLow (noisy)Large datasets
Mini-Batch GDSmall subsetMediumMediumLarge datasets (default choice)
AdamMini-batchVery fastHighDeep learning, very large datasets

7. Conclusion

Gradient Descent is the backbone of training in most machine learning and deep learning models. It minimises the cost function by iteratively adjusting model parameters in the direction of steepest descent. The choice of variant depends on dataset size, required convergence speed, and computational constraints. In practice, Adam or Mini-Batch SGD are the most commonly used variants due to their balance of speed and stability.

Question 26

Discuss about the Naïve Bayes classifier in detail.

Naïve Bayes is a probabilistic classification algorithm based on Bayes' Theorem. It assumes that all features are conditionally independent of each other given the class label. Despite this simplifying assumption, the algorithm performs well in practice, particularly for text classification and spam filtering tasks. It computes the posterior probability of each class given the input features and selects the class with the highest probability as the prediction.

1. Bayes' Theorem

The mathematical foundation of Naïve Bayes is:

P(C | X) = [ P(X | C) · P(C) ] / P(X)
TermNameMeaning
P(C | X)Posterior probabilityProbability of class C given input data X
P(X | C)LikelihoodProbability of observing X if the class is C
P(C)Prior probabilityInherent probability of class C in the dataset
P(X)EvidenceTotal probability of X; constant across all classes

2. The Naïve Independence Assumption

Naïve Bayes assumes that all features x₁, x₂, …, xₙ are conditionally independent given the class label C. This allows the likelihood to be decomposed as a product of individual feature likelihoods:

P(X | C) = P(x₁ | C) · P(x₂ | C) · ... · P(xₙ | C)

In practice, this assumption is rarely strictly true. For example, in email spam detection, the words "offer" and "free" are treated as independent even though they frequently co-occur. Nonetheless, the assumption greatly simplifies computation and still yields competitive accuracy.

3. Working of Naïve Bayes

  1. Calculate the prior probability P(C) for each class from the training data.
  2. Calculate the likelihood P(xᵢ | C) for each feature value given each class.
  3. Multiply likelihoods across all features to obtain P(X | C).
  4. Apply Bayes' theorem to compute the posterior P(C | X) for each class.
  5. Select the class with the highest posterior probability as the predicted label.

4. Types of Naïve Bayes Classifiers

TypeData TypeAssumptionCommon Use Case
Gaussian Naïve BayesContinuousFeatures follow a normal (Gaussian) distributionMedical diagnosis, iris classification
Multinomial Naïve BayesDiscrete (counts)Features represent event frequencies or word countsText classification, document categorisation
Bernoulli Naïve BayesBinary (0 or 1)Each feature is a binary indicator of presence or absenceSpam detection using word presence/absence flags

5. Advantages and Limitations

AdvantagesLimitations
Computationally efficient; fast to train and predictThe independence assumption is often unrealistic
Performs well with small training datasetsCannot capture feature correlations or interactions
Scales well to high-dimensional dataSensitive to zero-frequency problem (requires smoothing)
Well-suited for text and document classificationNot suitable for regression or complex non-linear problems

6. Applications

  • Email spam filtering
  • Sentiment analysis
  • News article classification
  • Medical diagnosis systems
  • Recommendation systems

7. Conclusion

Naïve Bayes is a simple yet effective probabilistic classifier that leverages Bayes' theorem with the assumption of conditional feature independence. It is particularly suitable for text-based and high-dimensional classification problems, offering fast training, low computational cost, and reasonable accuracy. It remains a foundational algorithm in machine learning despite the naïvety of its core assumption.

Question 27

Explain the multi-label classification methods and techniques in detail.

Multi-label classification is a type of supervised learning problem in which each instance can be assigned to multiple class labels simultaneously. This is in contrast to single-label classification, where each instance belongs to exactly one class. The output of a multi-label model is a set of labels rather than a single label.

1. Characteristics of Multi-label Problems

  • Each instance is associated with a set of labels rather than a single label.
  • Labels may be correlated or independent of one another.
  • The output space is large and potentially exponential in size.
  • Standard single-label evaluation metrics do not apply directly; specialised metrics are required.

2. Problem Transformation Methods

These methods convert a multi-label problem into one or more standard single-label classification problems, allowing conventional algorithms to be applied.

(a) Binary Relevance (BR)

Binary Relevance decomposes the multi-label problem into a set of independent binary classification tasks, one per label. Each binary classifier independently predicts whether its corresponding label is present or absent. The final predicted label set is the union of all positive predictions.

  • Advantages: Simple to implement; computationally scalable to large label sets.
  • Disadvantages: Ignores dependencies and correlations between labels, which can reduce accuracy when labels are related.

(b) Classifier Chains (CC)

Classifier Chains extends Binary Relevance by arranging the binary classifiers in a sequential chain. Each classifier receives the original feature vector along with the binary predictions of all preceding classifiers as additional input features. This allows the model to capture inter-label dependencies by conditioning each prediction on the outcomes of prior classifiers in the chain.

  • Advantages: Explicitly models label correlations; generally outperforms BR when labels are dependent.
  • Disadvantages: Errors made by earlier classifiers propagate through the chain; performance depends on the ordering of classifiers.

(c) Label Powerset (LP)

Label Powerset treats each unique combination of labels present in the training data as a distinct class, converting the multi-label problem into a standard multi-class classification problem over label subsets.

  • Advantages: Fully captures label correlations, as label combinations are modelled holistically.
  • Disadvantages: The number of distinct classes grows exponentially with the number of labels; cannot predict label combinations not seen during training.

3. Algorithm Adaptation Methods

Rather than transforming the problem, these methods modify existing classification algorithms to natively handle multi-label output.

(a) ML-kNN (Multi-Label k-Nearest Neighbours)

ML-kNN adapts the k-Nearest Neighbours algorithm for multi-label settings. For a new instance, the k nearest neighbours are identified. The algorithm uses the label frequencies among those neighbours along with Bayesian priors to estimate the posterior probability for each label. Labels whose posterior probability exceeds a set threshold are predicted as positive.

(b) RAKEL (RAndom k-labELsets)

RAKEL randomly partitions the full label set into small overlapping subsets. A Label Powerset classifier is trained on each subset. During prediction, the outputs of all subset classifiers are aggregated by voting to produce the final label set. This approach manages the exponential label-combination problem of LP while partially preserving label dependencies.

(c) Deep Learning Models

Neural networks are highly suitable for multi-label classification. The output layer is configured with one neuron per label, each with a sigmoid activation function so that each output is an independent probability in [0, 1]. Labels whose predicted probability exceeds a threshold (typically 0.5) are assigned as positive. The model is trained using binary cross-entropy loss computed independently for each label.

4. Evaluation Metrics

MetricDescription
Hamming LossFraction of labels incorrectly predicted (false positives and false negatives) averaged across all instances.
Subset AccuracyProportion of instances for which the predicted label set exactly matches the true label set; the strictest metric.
Precision / Recall / F1Standard metrics extended to multi-label settings, computed per label and then macro- or micro-averaged.
Jaccard IndexIntersection of predicted and true label sets divided by their union.

5. Applications

  • Image tagging (detecting multiple objects in a single image)
  • Text categorisation (assigning multiple topics to a document)
  • Music genre classification
  • Medical diagnosis (a patient may present multiple co-occurring conditions)
  • Video scene labelling

6. Conclusion

Multi-label classification is an important paradigm for problems where instances belong to several categories simultaneously. Methods such as Binary Relevance, Classifier Chains, and Label Powerset offer different trade-offs between simplicity and the ability to model label dependencies. Algorithm adaptation approaches such as ML-kNN and deep learning models provide powerful alternatives. The choice of method depends on the number of labels, the degree of inter-label dependency, and the available computational resources.

Question 28

Describe multi-class classification in detail.

Multi-class classification is a type of supervised learning in which a model must assign each input instance to one of three or more mutually exclusive classes. Unlike binary classification, which involves only two classes, and multi-label classification, where multiple labels may be assigned simultaneously, each data point in a multi-class problem belongs to exactly one class.

1. Comparison of Classification Types

TypeNumber of ClassesLabels per InstanceExample
Binary2Exactly 1Spam / Not Spam
Multi-class> 2 (mutually exclusive)Exactly 1Classify fruit as apple, banana, or orange
Multi-label> 2 (non-exclusive)One or moreA movie tagged as Action, Comedy, and Thriller

2. Techniques for Multi-class Classification

(a) One-vs-All (OvA) / One-vs-Rest (OvR)

For a dataset with N classes, N separate binary classifiers are trained. Each classifier learns to distinguish one class from all remaining classes combined. For a new instance, all N classifiers produce a confidence score and the class whose classifier produces the highest score is selected as the prediction.

Example with three classes — Green, Blue, Red (N = 3):

  • Classifier 1: [Green] vs [Red, Blue]
  • Classifier 2: [Blue] vs [Green, Red]
  • Classifier 3: [Red] vs [Blue, Green]
Number of classifiers = N

(b) One-vs-One (OvO)

A separate binary classifier is trained for every unique pair of classes. A new instance is passed through all classifiers; each votes for one of its two classes, and the class receiving the most votes overall is the final prediction.

Using the same three-class example (N = 3):

  • Classifier 1: Green vs Blue
  • Classifier 2: Green vs Red
  • Classifier 3: Blue vs Red
Number of classifiers = N × (N − 1) / 2  →  for N=3: 3 classifiers
AspectOne-vs-All (OvA)One-vs-One (OvO)
Classifiers requiredNN(N−1)/2
Training data per classifierFull dataset (class imbalance possible)Subset of two classes only
Prediction methodHighest confidence scoreMajority vote
Preferred whenN is largeEach binary subproblem is simpler

(c) Softmax Regression (Multinomial Logistic Regression)

Softmax regression generalises logistic regression directly to the multi-class setting without binary decomposition. It computes a probability distribution over all K classes simultaneously using the softmax function:

P(y = k | x) = exp(zₖ) / Σⱼ exp(zⱼ)

where zₖ is the linear score for class k. The class with the highest probability is selected as the prediction. Softmax regression is widely used as the output layer of neural networks for multi-class classification tasks, and is trained using categorical cross-entropy loss.

3. Common Algorithms

  • Decision Trees and Random Forests — natively multi-class
  • k-Nearest Neighbours — natively multi-class
  • Support Vector Machines — extended via OvA or OvO decomposition
  • Logistic Regression — extended via Softmax
  • Neural Networks — Softmax output layer with categorical cross-entropy

4. Conclusion

Multi-class classification extends binary classification to problems involving three or more mutually exclusive categories. Techniques such as One-vs-All, One-vs-One, and Softmax regression provide different strategies for managing the increased complexity of multiple classes. With appropriate algorithm selection, feature engineering, and evaluation, multi-class classification can be applied effectively to a wide range of real-world problems including handwritten digit recognition, image categorisation, and natural language processing.

1. Logistic Regression

Logistic Regression is a supervised machine learning algorithm used primarily for binary classification problems, where the dependent variable takes one of two possible values (e.g., 0 or 1, Pass or Fail). Unlike linear regression, which produces unbounded continuous outputs, logistic regression predicts the probability that a given input belongs to a particular class, ensuring the output remains within the valid range of [0, 1].

The model establishes a linear combination of input features, which is then passed through the sigmoid (logistic) function to produce a probability. A decision threshold — typically 0.5 — is applied to convert the probability into a discrete class label.

2. The Sigmoid Function

The sigmoid function is defined as:

σ(z) = 1 / (1 + e−z)

where z = β₀ + β₁x₁ + β₂x₂ + … is the linear combination of input features and model parameters (weights).

Question 30

Discuss about Precision, Recall, and their Trade-offs in detail.

Precision and Recall are two fundamental evaluation metrics used in classification problems, particularly when the dataset is imbalanced or when the costs associated with different types of misclassification are unequal. Both metrics are derived from the confusion matrix, which records the counts of True Positives (TP), True Negatives (TN), False Positives (FP), and False Negatives (FN).

1. Precision

Precision measures the proportion of instances predicted as positive that are actually positive. It answers the question: Of all the instances labelled positive by the model, how many were correct?

Precision = TP / (TP + FP)

A high precision value indicates that the model produces few false positives — when it predicts the positive class, it is likely to be correct. This property is critical in applications such as spam detection, where incorrectly classifying a legitimate email as spam is undesirable.

Example: In a spam detection system, if 100 emails are predicted as spam and 80 are actually spam: Precision = 80 / 100 = 0.80 (80%).

2. Recall (Sensitivity)

Recall measures the proportion of actual positive instances that were correctly identified by the model. It answers the question: Of all instances that are truly positive, how many did the model successfully identify?

Recall = TP / (TP + FN)

A high recall value indicates that the model misses few actual positive cases. This is critical in applications such as medical diagnosis, where failing to detect a disease (a false negative) can have serious consequences.

Example: In a disease detection system with 100 sick patients, if the model correctly identifies 90: Recall = 90 / 100 = 0.90 (90%).

3. Precision vs. Recall — Key Differences

MetricFocusMinimisesCritical When
PrecisionCorrectness of positive predictionsFalse Positives (FP)False alarms are costly

4. The Precision–Recall Trade-off

In most classification systems, precision and recall cannot be simultaneously maximised. This trade-off arises from the decision threshold used to classify instances:

  • Lowering the decision threshold causes the model to predict the positive class more liberally, increasing recall (fewer true positives are missed) but decreasing precision (more false positives are introduced).
  • Raising the decision threshold causes the model to predict the positive class more conservatively, increasing precision (fewer false positives) but decreasing recall (more true positives are missed).

Scenario 1 — High Precision, Low Recall: The model is conservative and only predicts positive when highly certain. This results in few false positives but many missed actual positives. Typical use case: fraud detection, where falsely flagging a legitimate transaction is costly.

Scenario 2 — High Recall, Low Precision: The model is lenient and predicts positive broadly. This captures most actual positives but introduces many false alarms. Typical use case: cancer screening, where missing a positive diagnosis is dangerous.

5. F1 Score — Balancing Both Metrics

The F1 Score is the harmonic mean of Precision and Recall, providing a single metric that balances both concerns. It is especially useful when there is an unequal class distribution or when both false positives and false negatives carry significant costs.

F1 = 2 × (Precision × Recall) / (Precision + Recall)

The harmonic mean penalises extreme imbalances between precision and recall more severely than the arithmetic mean, making it a reliable balanced metric.

7. Conclusion

Precision and Recall are complementary metrics that capture different aspects of a classifier's performance. Precision reflects how reliable positive predictions are, while Recall reflects how completely the model identifies all actual positive instances. Because improving one typically reduces the other, practitioners must select the appropriate metric — or use the F1 Score as a balanced measure — based on the specific costs and risks associated with false positives and false negatives in the given application domain.
