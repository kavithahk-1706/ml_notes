Question 11

Explain in detail about Regression Analysis and mention the importance of the measure of linear relationship.

1. Definition of Regression Analysis

Regression Analysis is a statistical process used to estimate the relationships between a dependent variable (also called the criterion or response variable) and one or more independent variables (also called predictors or features). It is used to understand how the dependent variable changes when any of the independent variables is varied, and to predict the value of the dependent variable for given values of the predictors.

3. Measure of Linear Relationship

Measuring the strength and direction of the linear relationship between variables is central to regression analysis. The key measures are:

(a) Linear Regression as a Measure

Linear regression directly quantifies the relationship between a dependent and independent variable. It shows how the response changes per unit change in the predictor. There is a risk of overfitting — the model may perform well on training data but poorly on new data if the relationship is forced to be linear when it is not.

Y' = bX + A

where b is the slope (rate of change) and A is the y-intercept (value of Y when X = 0).

(b) Polynomial Regression

Used when data shows a curved pattern. It extends linear regression by adding higher-degree terms, allowing the model to capture non-linear relationships while still using the least squares fitting method.

(c) Ridge Regression

Used when multiple predictors are highly correlated (multicollinearity problem). Adds a small bias (L2 penalty) to reduce variance and stabilise the coefficient estimates.

(d) Lasso Regression

Performs both variable selection and regularisation simultaneously. Uses soft thresholding — coefficients of less important variables are shrunk to exactly zero, effectively removing them from the model.

(e) ElasticNet Regression

Combines both Ridge (L2) and Lasso (L1) penalties. Useful when there are many correlated features and variable selection is desired. Applied in support vector machines and portfolio optimisation.

(f) Stepwise Regression

Automatically selects the most useful subset of predictors. Three methods:

  • Forward selection: Start with no variables; add the most significant one at each step.
  • Backward elimination: Start with all variables; remove the least significant one at each step.
  • Bidirectional elimination: Combination of both — variables can be added or removed at each step.

4. Conclusion

Regression analysis is a powerful tool for modelling and predicting relationships between variables. The measure of linear relationship — through the regression equation, coefficients, and correlation metrics — quantifies how strongly and in what direction variables are related. Choosing the right type of regression (linear, polynomial, regularised, stepwise) depends on the nature of the data, the presence of multicollinearity, and the need for variable selection.

Question 12

Discuss the significance of coefficients in Linear Regression and compute the coefficient and Y-intercept for given values using the least squares method.

1. Linear Regression — Recap

Simple linear regression models the relationship between one independent variable (X) and one dependent variable (Y) using a straight-line equation:

Y' = bX + A

where b is the regression coefficient (slope) and A is the Y-intercept.

2. Significance of the Coefficient (b — Slope)

  • The slope b represents the rate of change in Y for a one-unit increase in X.
  • It quantifies the strength and direction of the relationship:
    • b > 0 → positive relationship (Y increases as X increases)
    • b < 0 → negative relationship (Y decreases as X increases)
    • b = 0 → no linear relationship between X and Y
  • A larger |b| indicates a steeper line and a stronger effect of X on Y.
  • The coefficient is estimated from data to minimise prediction error.

3. Significance of the Y-Intercept (A)

  • The Y-intercept A is the predicted value of Y when X = 0.
  • It positions the regression line on the Y-axis.
  • In some contexts it has a direct practical interpretation; in others it is purely mathematical.

4. Least Squares Method

The least squares method finds the values of b and A that minimise the sum of squared residuals (SSR) — the total squared difference between observed values and predicted values:

SSR = Σ (Yᵢ − Ŷᵢ)²

The formulas derived from this minimisation are:

b = [n·ΣXY − ΣX·ΣY] / [n·ΣX² − (ΣX)²]
A = (ΣY − b·ΣX) / n = Ȳ − b·X̄

5. Worked Example

Given the following dataset:

X (Hours studied)Y (Marks)X²XY
1212
2346

ΣX = 15ΣY = 20ΣX² = 55ΣXY = 69

n = 5

Step 1 — Calculate b (slope):

b = (5 × 69 − 15 × 20) / (5 × 55 − 15²) = (345 − 300) / (275 − 225) = 45 / 50 = 0.9

Step 2 — Calculate A (y-intercept):

A = (20 − 0.9 × 15) / 5 = (20 − 13.5) / 5 = 6.5 / 5 = 1.3

Regression Equation:

Y' = 0.9X + 1.3

Interpretation: For every additional hour studied, marks increase by 0.9. A student who studies 0 hours is predicted to score 1.3 marks.

6. Conclusion

The regression coefficient (slope) quantifies the effect of the independent variable on the dependent variable, while the Y-intercept fixes the baseline prediction. The least squares method provides the mathematically optimal estimates by minimising total squared prediction error, producing the best-fit straight line through the data.

Question 13

Discuss the different performance metrics of regression models.

Performance metrics for regression models measure how accurately the model predicts continuous output values. Unlike classification metrics, regression metrics focus on the magnitude of prediction errors — the difference between actual values (Yᵢ) and predicted values (Ŷᵢ).

1. Mean Absolute Error (MAE)

MAE calculates the average of the absolute differences between actual and predicted values. It gives an intuitive measure of average prediction error in the same unit as the output variable.

MAE = (1/n) × Σ |Yᵢ − Ŷᵢ|
  • All errors are weighted equally regardless of magnitude.
  • Less sensitive to outliers than MSE.
  • A lower MAE indicates better model performance.

2. Mean Squared Error (MSE)

MSE calculates the average of the squared differences between actual and predicted values. Squaring the errors gives proportionally more weight to large errors.

MSE = (1/n) × Σ (Yᵢ − Ŷᵢ)²
  • More sensitive to outliers than MAE — a single large error inflates MSE significantly.
  • Differentiable everywhere, making it suitable for gradient-based optimisation.
  • Commonly used as the loss function during model training.

3. Root Mean Squared Error (RMSE)

RMSE is the square root of MSE, restoring the metric to the same unit as the output variable.

RMSE = √[ (1/n) × Σ (Yᵢ − Ŷᵢ)² ]
  • More interpretable than MSE because it is on the same scale as Y.
  • Still sensitive to outliers (due to the squaring in MSE).
  • Widely used as the primary evaluation metric in regression tasks.

4. R-squared (R²) — Coefficient of Determination

R² measures the proportion of variance in the dependent variable that is explained by the model's predictions.

R² = 1 − (SS_residual / SS_total)

where SS_residual = Σ(Yᵢ − Ŷᵢ)² and SS_total = Σ(Yᵢ − Ȳ)².

  • Range: 0 to 1. R² = 1 → perfect fit; R² = 0 → model explains none of the variance.
  • Higher R² indicates a better fit.
  • Does not account for the number of predictors — can be artificially inflated by adding irrelevant variables.

5. Adjusted R-squared

Adjusted R² improves on R² by penalising the addition of unnecessary independent variables. It only increases if a new variable genuinely improves the model.

Adj. R² = 1 − [(1 − R²)(n − 1) / (n − p − 1)]

where n is the number of observations and p is the number of predictors.

  • Preferred over R² when comparing models with different numbers of predictors.
  • Helps avoid overfitting in multiple regression.

6. Mean Absolute Percentage Error (MAPE)

MAPE expresses prediction error as a percentage of the actual values, making it scale-independent.

MAPE = (100/n) × Σ | (Yᵢ − Ŷᵢ) / Yᵢ |
  • Intuitive — tells you the average percentage error of predictions.
  • Undefined when any actual value Yᵢ = 0.

7. Explained Variance Score

Measures how much of the total variance in the data is captured by the model's predictions.

Explained Variance = 1 − Var(Y − Ŷ) / Var(Y)
  • A higher value indicates that the model captures more of the data's variability.
  • Similar to R² but not identical — differs when the mean of residuals is non-zero.

8. Summary Table

R²UnitlessModerate1Relative model fit; easy to compare

9. Conclusion

Choosing the right regression performance metric depends on the application. MAE is robust to outliers and easy to interpret. RMSE is standard and widely reported but is sensitive to large errors. R² and Adjusted R² measure how well the model explains the variance in the data. MAPE is useful when a percentage-based error interpretation is needed. In practice, multiple metrics are often reported together for a complete picture of model quality.

Question 14

Discuss the significance of coefficients in Linear Regression and compute the coefficient and Y-intercept for the given values.

This question combines the theory of coefficient significance with a computational problem. The least squares method is used to determine the best-fit line.

1. Significance of Regression Coefficients

In a simple linear regression model Y' = bX + A:

  • Slope (b): Measures the change in Y for each unit increase in X. Indicates the strength and direction of the linear relationship. A statistically significant b means X is a meaningful predictor of Y.
  • Y-intercept (A): The predicted value of Y when X = 0. Anchors the regression line on the Y-axis.
  • Testing whether b ≠ 0 (via t-test or F-test) confirms whether the predictor has a statistically significant effect on the response.
  • The sign of b tells whether the relationship is positive or negative; the magnitude tells how large the effect is.

2. Least Squares Formulas

b = [n·ΣXY − ΣX·ΣY] / [n·ΣX² − (ΣX)²]
A = Ȳ − b·X̄ = (ΣY − b·ΣX) / n

3. Worked Example

Given data (advertising spend X vs sales Y):

XYX²XY
210420
4201680
62536150
83064240
1040100400
ΣX = 30ΣY = 125ΣX² = 220ΣXY = 890

n = 5

Compute slope b:

b = (5 × 890 − 30 × 125) / (5 × 220 − 30²) = (4450 − 3750) / (1100 − 900) = 700 / 200 = 3.5

Compute intercept A:

A = (125 − 3.5 × 30) / 5 = (125 − 105) / 5 = 20 / 5 = 4.0

Regression Equation:

4. Conclusion

The regression coefficient b and intercept A derived using the least squares method provide the optimal straight-line fit through the data. The coefficient is the most important parameter — it quantifies the linear effect of the predictor on the response and is the basis for inferential testing of whether that relationship is statistically meaningful.

Question 15

Differentiate between simple linear regression and multiple linear regression models with examples.

1. Simple Linear Regression (SLR)

Simple linear regression models the relationship between one independent variable and one dependent variable. The relationship is assumed to be a straight line.

Y = β₀ + β₁X + ε
  • β₀ = y-intercept; β₁ = slope; ε = error term
  • Easy to understand, visualise (2D plot), and compute.
  • Suitable when only one variable significantly drives the output.

Example: Predicting salary based on years of experience. One input (experience), one output (salary).

2. Multiple Linear Regression (MLR)

Multiple linear regression extends SLR to two or more independent variables. Each predictor has its own coefficient representing its individual contribution to the output.

Y = β₀ + β₁X₁ + β₂X₂ + … + βₙXₙ + ε
  • More complex — the model is a hyperplane in n-dimensional space rather than a line.
  • Captures the combined effect of multiple factors on the dependent variable.
  • Higher accuracy when the outcome is influenced by several variables simultaneously.

Example: Predicting house price based on area, number of rooms, location, and age of building. Multiple inputs, one output.

Ease of understandingEasy to understand and calculateMore complex
ComputationLess computation requiredMore computation required

4. Conclusion

Simple linear regression is the appropriate choice when one predictor is sufficient to explain the output and simplicity is valued. Multiple linear regression is necessary when the outcome depends on several interacting factors and higher predictive accuracy is required. The choice between them depends on the problem domain, the available data, and the desired balance between interpretability and accuracy.

Question 16

Explain the need for polynomial regression in detail.

1. Definition

Polynomial regression is an extension of linear regression used when the relationship between the independent variable and the dependent variable is non-linear (curved) rather than a straight line. It fits a polynomial equation to the data using the least squares method. Unlike simple linear regression, the best-fit line is a curve rather than a straight line.

Polynomial regression is technically a variant of multiple linear regression, as the polynomial terms (x², x³, etc.) can be treated as additional independent variables.

Y = a₀ + a₁x + a₂x² + a₃x³ + … + aₙxⁿ

2. Need for Polynomial Regression

(a) Limitation of Linear Regression

Linear regression assumes a straight-line relationship between variables. In many real-world scenarios, however, the data follows a curved or non-linear pattern. Forcing a straight line through such data produces large errors and poor predictions. Polynomial regression overcomes this by allowing the model to bend to fit the data's actual shape.

(b) Handles Curved (Non-linear) Data

When a scatter plot of the data shows a clear curve — such as an upward-accelerating or U-shaped trend — polynomial regression can capture this by adding higher-degree terms (x², x³, …) to the model equation. These terms introduce the flexibility needed to follow the curve.

(c) Better Accuracy

Since polynomial regression fits the actual shape of the data more closely, it produces more accurate predictions than linear regression when the underlying relationship is non-linear. The model's residuals (errors) are significantly smaller.

(d) Models Complex Relationships

Many real-world phenomena — population growth, drug response curves, economic trends — follow complex non-linear patterns that no straight line can adequately model. Polynomial regression provides a principled way to fit such patterns.

(e) Flexible Model

The degree n of the polynomial can be adjusted to match the complexity of the data pattern. A degree-2 polynomial (parabola) fits simple curves; higher degrees can fit more complex shapes. This flexibility makes the model adaptable to diverse datasets.

(f) Uses the Least Squares Method

Like linear regression, polynomial regression is fitted using the least squares method, which minimises the sum of squared differences between observed and predicted values. The underlying optimisation is the same; only the model structure changes.

(g) Real-World Applications

  • Growth rate analysis (biological or economic growth curves)
  • Population prediction over time
  • Temperature and weather forecasting
  • Physics — projectile motion, acceleration curves
  • Finance — modelling non-linear price trends

3. Risk — Overfitting

A key limitation of polynomial regression is the risk of overfitting when the polynomial degree is too high. The model may fit the training data very closely — including noise — but perform poorly on new data. Cross-validation and regularisation are used to manage this risk.

4. Conclusion

Polynomial regression is essential when real-world data exhibits non-linear patterns that a simple straight line cannot capture. By incorporating higher-degree terms of the independent variable, it provides the flexibility to model curves accurately while still using the familiar and well-understood least squares optimisation framework. The appropriate polynomial degree should be selected carefully to balance accuracy and the risk of overfitting.

Question 18

Discuss about overfitting in detail. List the reasons for overfitting and ways to tackle overfitting.

1. Definition

Overfitting occurs when a machine learning model learns the training data too well — including its noise and random fluctuations — to the extent that it performs very well on training data but generalises poorly to new, unseen data. The model has effectively memorised the training examples rather than learning the underlying pattern.

Overfitting is characterised by low bias and high variance: the model is very flexible and fits training data tightly, but its predictions vary wildly on new data.

2. Analogy

A student who memorises every answer from past papers without understanding the concepts may score perfectly on those papers but fail in the actual exam when questions are rephrased. This is analogous to an overfitted model.

3. Reasons for Overfitting

ReasonExplanation
Overly complex modelToo many parameters, layers, or polynomial degree — the model learns noise as if it were signal.
Small training datasetInsufficient data forces the model to memorise the few examples it has rather than generalise.
Too many featuresIrrelevant features add noise and increase model complexity unnecessarily.
High-degree polynomialHigh-degree curves pass through every training point, including outliers and noise.
Training for too many epochsExcessive training causes the model to over-learn the training set; validation error starts increasing while training error keeps falling.
Noisy dataErrors and random fluctuations in the training data mislead the model into learning false patterns.
Lack of regularisationWithout any penalty on model complexity, the algorithm freely fits any pattern it finds.

4. Ways to Tackle Overfitting

TechniqueHow It Helps
Increase training dataMore diverse examples help the model learn genuine patterns rather than memorising specific instances.
Simplify the modelReduce parameters, layers, or polynomial degree — a simpler model has less capacity to overfit.
Feature selectionRemove unnecessary or irrelevant features to reduce complexity and noise.
Regularisation (Ridge / Lasso)Adds a penalty term to the loss function that discourages large coefficient values, constraining model flexibility.
Cross-validationk-fold cross-validation evaluates model performance on multiple held-out subsets, providing a more reliable estimate of generalisation.
Early stoppingMonitor validation error during training; stop when it begins to increase, preventing the model from over-learning the training set.
Dropout (Neural Networks)Randomly deactivates neurons during training, preventing co-adaptation and improving generalisation.
Data augmentationArtificially expand the training set by applying transformations (rotation, noise, flipping) to existing examples.
Reduce noise in dataClean and preprocess data properly to remove errors and outliers that the model might learn as patterns.

5. Conclusion

Overfitting is one of the most common and important challenges in machine learning. A model that overfits may appear highly accurate during training but fails in real-world deployment. Prevention requires a combination of strategies — collecting more data, simplifying the model, applying regularisation, and validating performance on held-out data. The goal is always to find the right balance between fitting the training data and generalising to new examples.

Question 19

Differentiate between simple linear regression and polynomial regression models with examples.

1. Simple Linear Regression

Simple linear regression models the relationship between one independent variable and one dependent variable as a straight line.

Y = β₀ + β₁X + ε
  • Assumes a linear (first-degree) relationship.
  • Simple, fast to compute, and easy to interpret.
  • Appropriate when data follows a linear trend.
  • Low risk of overfitting.

Example: Predicting salary from years of experience — each additional year adds a roughly constant increment to salary.

2. Polynomial Regression

Polynomial regression uses one independent variable but includes higher-degree terms (X², X³, …) to model a curved (non-linear) relationship.

Y = a₀ + a₁X + a₂X² + a₃X³ + … + aₙXⁿ + ε
  • Models non-linear (curved) relationships.
  • More flexible than simple linear regression.
  • Appropriate when a scatter plot shows a curved trend.
  • Higher risk of overfitting as degree increases.

Example: Predicting population growth over time — early growth is slow, then accelerates, then slows again, forming an S-curve or parabola.

  • Use simple linear regression when the scatter plot shows a roughly straight-line trend and interpretability is important.

Simple linear and polynomial regression are closely related — polynomial regression is literally an extension of linear regression with added terms. The key difference is flexibility: polynomial regression can capture curves that linear regression cannot. However, this flexibility comes at the cost of increased overfitting risk. Selecting the appropriate model type depends on the shape of the data distribution, the available training set size, and the desired balance between accuracy and generalisability.

Question 20

Discuss about underfitting in detail. List the reasons for underfitting and ways to tackle underfitting.

1. Definition

Underfitting is a condition in machine learning where a model is too simple to capture the underlying patterns in the data. An underfitted model performs poorly on both training data and test data — it has not learned enough from the training examples to make useful predictions.

Underfitting is characterised by high bias and low variance: the model consistently makes the same types of errors regardless of which data it is applied to, because it lacks the capacity to represent the true underlying function.

2. Reasons for Underfitting

ReasonExplanation
Model is too simpleUsing a linear model for data with a non-linear pattern means the model structurally cannot fit the true relationship. Example: applying linear regression to data that follows a quadratic trend.
Insufficient trainingTraining for too few epochs or iterations prevents the model from fully adjusting its parameters. The model stops learning before convergence.
Missing important featuresKey variables that influence the output are absent from the dataset. Without the right information, the model cannot learn accurately.
Excessive regularisationToo-large regularisation penalty (λ) forces coefficients so close to zero that the model cannot even fit the training data properly.
Poor feature engineeringRaw features may not represent the problem well. Without appropriate transformations or derived features, even a complex model may underfit.
Very small datasetWith extremely little data, the model may not have enough examples to learn any meaningful pattern.

3. Ways to Tackle Underfitting

TechniqueHow It Helps
Increase model complexityUse a more powerful model — polynomial regression instead of linear, a deeper neural network, or a decision tree with greater depth. Allows the model to capture non-linear patterns.
Add relevant featuresInclude more informative input variables. Feature engineering (creating new features from existing ones) can significantly improve learning.
Reduce regularisationLower the regularisation parameter λ to give the model more freedom to fit the training data.
Train for more epochsAllow the model more iterations to continue adjusting parameters and minimising training error.
Use non-linear modelsReplace linear models with algorithms that can capture complex, non-linear relationships — decision trees, SVMs with RBF kernel, neural networks.
Improve data qualityClean the dataset — remove noise, correct errors, handle missing values. High-quality data enables better learning even with simple models.
Reduce learning rate carefullyA very high learning rate can cause the model to skip over the optimal parameters. Tuning the learning rate allows more careful convergence.

4. Overfitting vs. Underfitting — Comparison

AspectUnderfittingOverfitting
BiasHighLow
VarianceLowHigh
Training errorHighVery low
Test errorHighHigh
Model complexityToo simpleToo complex
Fix directionIncrease complexityDecrease complexity

5. Conclusion

Underfitting results from a model that is too constrained to capture the true patterns in data. It manifests as high error on both training and test sets — the model is not useful for either. Solutions involve increasing model capacity, improving features, and reducing over-regularisation. The ideal model sits between underfitting and overfitting — achieving the right balance between bias and variance for best generalisation performance. This balance point is known as the bias-variance tradeoff.
